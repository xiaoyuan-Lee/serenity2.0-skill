# Serenity 2.0 × A股：卡脖子选股框架

## 使用方式

在对话中输入以下命令：

| 命令 | 说明 |
|------|------|
| `/serenity-a-share analyze <产业链>` | 深度分析一个产业链 |
| `/serenity-a-share quick <产业链/标的>` | 速通模式（5分钟快速判断） |
| `/serenity-a-share review <标的>` | 复盘已有持仓标的 |
| `/serenity-a-share status` | 查看当前所有追踪的产业链/标的 |

> 如果不带子命令直接调用 `/serenity-a-share`，显示本帮助菜单。

## 执行流程

### 0. 准备工作

第一步：读取以下框架文件到上下文（必须在执行任何分析前完成）：

1. `framework/serenity-a-share-strategy.md` — 主框架（策略+仓位管理+迭代机制）
2. `framework/bom-analysis-protocol.md` — BOM拆解标准执行协议（评分标准+辩论机制）

第二步：检查 `~/.claude/projects/C--Users-l1592/memory/.serenity/memory/` 下是否存在该产业链的历史分析记录。

### 1. analyze 模式

对于 `analyze` 命令，按**螺旋迭代架构**执行，而非瀑布流：

```
螺旋1（宽筛）：政策前置过滤 → 确认是否值得继续
螺旋2（初评）：BOM表层拆解，只拆到子系统级
螺旋3（精选）：三高快评（公开数据），过滤<3.5的环节
螺旋4（深评）：完整BOM+辩论+验证，出最终结论
```

每个螺旋末尾执行GO/NO-GO门控检查：
- 政策评级⭐ → 直接终止该链
- BOM无上市公司 → 终止
- 三高评分无≥3.5环节 → 终止
- 辩论后置信度"低" → 标记待验证

### 2. quick 模式

执行**速通核心清单**（3个信号）：

| # | 检查项 | 结论 |
|---|-------|------|
| ① 逻辑还在不在？ | 营收增速>15% + 政策评级未降 |
| ② 价格贵不贵？ | PS是否在估值区间内 |
| ③ 周期对不对？ | 板块成交额占比是否<70%百分位 |

三个🟢→维持不动。任意🔴→暂停操作，建议切analyze做深度分析。

### 3. review 模式

对已有持仓标的执行：
1. 前置过滤（流动性+筹码结构是否有变化）
2. 7项验证（5项基本面+流动性+筹码）
3. 三类退出逻辑检查（兑现/证伪/被替代）
4. 周期状态更新
5. 输出：维持/加减仓/清仓建议

### 4. status 模式

扫描memory中所有历史分析记录，输出：
- 当前追踪的产业链列表及其状态
- 每个产业链的周期判定
- 核心持仓的评分变化趋势

## 产出要求

所有分析输出必须包含：

1. **结构化数据**（三高评分表、评分详情）
2. **关键依据**（数据来源标注：审计级/券商级/媒体级）
3. **置信度标注**（高/中/低）
4. **风险提示**（辩论中发现的空头论点）
5. **仓位建议**（基于周期阶段+评分评级的明确动作）

## 记忆写入

- analyze完成 → 将分析报告写入 `~/.claude/projects/C--Users-l1592/memory/.serenity/memory/<产业链>/<日期>.md`
- review完成 → 追加到该标的的反思记录

## 速查

| 三高综合分 | 评级 | 动作 |
|-----------|------|------|
| ≥4.5 | 核心重仓区 | 哑铃核心仓位，优先配置 |
| 3.5-4.4 | 卫星配置区 | 哑铃卫星仓位，≤正常仓位60% |
| 2.5-3.4 | 观察名单 | 不入仓，跟踪 |
| <2.5 | 暂不考虑 | 跳过 |
