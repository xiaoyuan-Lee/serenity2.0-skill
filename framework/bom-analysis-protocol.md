# BOM深度拆解标准执行协议

> 当用户说"拆解[产业链名称]产业链"时，按此协议执行。

## 执行流程

### Phase 0：确认意图
- 用户提到的产业链名称是否明确？
- 是否有细分方向偏好（如"人形机器人"vs"工业机器人"）？
- 如有歧义，先询问澄清。

### Phase 1：宏观政策三级漏斗

**一级（国家战略）：** 产业链是否出现在以下文件中？
- 十五五规划 / 十四五规划纲要
- 国务院《中国制造2025》或类似顶层文件
- 中央经济工作会议、政治局集体学习明确点名

**二级（部委配套）：** 是否有以下配套？
- 工信部专项（如《人形机器人创新发展指导意见》）
- 发改委产业目录 / 产业结构调整指导目录
- 科技部重点研发计划专项

**三级（地方真金白银）：** 以下地方是否已投入？
- 合肥产投/建投、浙江"4+1"基金群
- 广东集成电路/新兴产业子基金
- 上海三大先导产业母基金
- 其他地方（北京、深圳、成都等）

**输出：** 政策评级（⭐⭐⭐主线 / ⭐⭐次线 / ⭐待观察）+ 政策依据摘要
- 主线：三级全部通过，或至少两级通过且明确点名
- 次线：仅通过一级到两级
- 待观察：未明确出现在任何政策中
⚠️ 待观察级直接终止，不在该链上继续投入分析时间。

### Phase 2：产品定义与拆解准备

1. **品类界定** — 终端产品是什么？与相似品类区别在哪？
2. **技术成熟度** — 实验室样品 / 量产早期 / 规模化放量
3. **中国卡位** — 中国在全产业链中的话语权（整体领先 / 部分领先 / 跟随 / 落后）
4. **标杆BOM参考** — 是否有海外或国内的典型产品的BOM结构可参考？

### Phase 3：递归式BOM拆解

按以下结构逐层拆解：

```
终端产品（100% BOM 成本）
  ├── 子系统1（成本占比 XX%）
  │    ├── 关键零部件1.1（占比 XX%，进口/国产状态）
  │    │    └── 上游材料（若关键则继续拆）
  │    ├── 关键零部件1.2
  │    └── ...
  ├── 子系统2
  └── ...
```

**拆解规则：**
- 只拆 **成本占比>5%** 或 **卡脖子程度🔴（完全进口）** 的环节
- 一般拆 2~3 层，最细到材料级（如磁钢、特种钢材、陶瓷基板）
- 每个节点标注：[成本占比] [国产化状态 🟢🟡🔴] [国内龙头（如有上市公司）]

### Phase 3.5：数据源标注（2026-07-03 新增）

每个评分数据必须标注来源与时效，防止不同来源口径混用导致评分偏移。

**标注格式：** 每个数据点括号标注 `[来源 | 截止时间 | 置信度]`

```
示例：
营收CAGR 47% [2025年报 | FY2025 | ✔️审计]
毛利率 36.77% [2025年报 | FY2025 | ✔️审计]
六维力传感器月出货千只 [华创证券研报 | 2026Q1 | ⚠️券商估算]
丝杠良率60% [行业媒体 | 2026Q1 | ⚠️非官方]
```

**来源分级：**

| 级别 | 定义 | 默认置信度 |
|------|------|----------|
| 🟢 审计级 | 上市公司年报/季报、交易所公告 | 高 |
| 🟡 券商/官方 | 券商研报、政府/协会数据、公司投资者关系 | 中 |
| 🔴 媒体/估算 | 行业媒体、自媒体、分析师模型推算 | 低（标注⚠️） |

**跨来源冲突处理：**
- 遇到数据打架时，取最低置信度来源的保守值
- 评分依据中注明"取保守值"及原因
- 在反思记录中记录该冲突，作为下次复查重点

#### 数据快照机制（Phase 3.5 — 优化新增）

> **问题：** BOM拆解过程中，同一个数据点可能在Phase 2（拆解）和Phase 4（评分）使用不同版本，导致评分标准不统一。
> **解决：** 每次分析开始时"冻结"所有引用数据版本。

**数据快照规则：**

1. **冻结时机：** Phase 2（拆解开始）时，一次性固定所有引用的财务数据和行业数据版本
2. **冻结内容：** 记录 `{数据点: 值, 来源: ..., 数据截止日期: ..., 冻结日期: ...}` 的快照
3. **冻结期间：** 整个分析周期内（Phase 2→Phase 6）不更新数据，确保内部一致性
4. **更新窗口：** 仅在分析完成后、下次分析开始时刷新

**数据时效性阈值（TLL—Time-to-Live）：**

| 数据类型 | 有效期限 | 过期后动作 |
|---------|---------|-----------|
| 季报/年报（审计级） | 至下一季报发布 | 继续使用，标注⚠️"非最新" |
| 券商研报（券商级） | 1个月 | 停止引用或标注"原始来源" |
| 媒体/行业数据（媒体级） | 1周 | 停止引用，标记为"已过期" |
| 国产化率估算 | 6个月 | 过期后必须重新搜索验证 |

**首次分析 vs 后续复查的差异：**
- 首次分析：全量快照 + 全量分析
- 复查：只拍摄新增/变化数据的快照，存量数据检查时效性
- 如果某关键数据点过期且无法更新 → 该环节置信度自动降一档

### Phase 4：三高评分（结构化输出）

对有上市公司的环节逐一评分，要求输出结构化数据而非自由文本，便于后续序列化复盘和时间序列对比。

#### 结构化Schema定义

每次评分输出遵循以下结构化格式：

```json
{
  "环节名称": "RV减速器",
  "所属子系统": "关节驱动系统",
  "三高评分": {
    "增长评分": { "值": 4.0, "依据": "营收CAGR 24%", "否决触发": false },
    "利润评分": { "值": 4.5, "依据": "毛利率52%", "否决触发": false },
    "围墙评分": { "值": 5.0, "依据": "技术+认证+专利三重封锁", "否决触发": false },
    "综合": 4.48,
    "评级": "卫星配置区"
  },
  "置信度": "高",
  "关键风险": "技术路线被谐波减速器替代",
  "对标海外": "纳博特斯克(Harmonic Drive)",
  "评分日期": "2026-07-03"
}
```

#### 评分标准

**① 高增长 — 营收CAGR（近3年）**
- ★★★★★ >30% | ★★★★☆ 20-30% | ★★★☆☆ 15-20% | ★★☆☆☆ 10-15% | ★☆☆☆☆ <10%
- ⚠️ 否决条件：营收CAGR<10% → **一票否决**（不管围墙多高，直接评为暂不考虑）
- ⚠️ 减速条件：连续2季营收增速环比下降 → 自动降一星
- ⚠️ 首次拆解无3年数据时：用上市以来复合增速代替，置信度自动降一档
- ⚠️ **新增：非有机增长折扣（优化改进）**
  - 并购贡献>30%营收增长的，CAGR打折系数×0.7（并购增长通常不可持续）
  - 周期性行业用3年营收**中位数增长率**替代CAGR（避免周期顶点虚高）
  - 次新股（上市<2年）置信度强制为"中"以下，CAGR使用中位数而非均值
- ⚠️ **新增：增长质量交叉验证**
  - 营收增速应同步伴随：①合同负债增长 ②经营性现金流增长 ③员工人数合理增长
  - 如果营收增速>30%但合同负债负增长 → 增长质量存疑，降一星

**② 高利润 — 毛利率 + 研发效率（优化改进）**
- ★★★★★ >60% | ★★★★☆ 45-60% | ★★★☆☆ 35-45% | ★★☆☆☆ 25-35% | ★☆☆☆☆ <25%
- ⚠️ 否决条件：毛利率<20% → **一票否决**
- ⚠️ 减速条件：连续3季毛利率下滑 → 降两星；**单季下滑>5个百分点** → 立即复查（不等3季）
- ⚠️ 扣非净利率连续2年<5%的标的，利润评分上限不超过★★★（防增收不增利）
- ⚠️ **新增：研发效率比率（优化改进）**
  - 毛利率/研发费用率 = 研发每投入1元能换回多少毛利保护
  - 研发效率>2.0 → 正常评分（研发投资有效率）
  - 研发效率1.0-2.0 → 利润评分自动降一星（研发投入效率不足）
  - 研发效率<1.0 → 利润评分上限★★（烧钱换技术，利润质量差）
  - **注意：** 此调整仅适用于研发费用率>15%的重研发公司，轻研发公司不适用
- ⚠️ **新增：毛利率绝对值和趋势双考核**
  - 毛利率50%→48%（-2pct）→ 🟡预警，标记观察
  - 毛利率50%→35%（-15pct）→ 🔴立即终止评分，重新评估护城河

**③ 高围墙 — 竞争壁垒（定性+定量锚点优化改进）**

**核心变革：** 原纯定性评分 → 引入"壁垒高度"和"壁垒持续性"两个子维度，并增加定量锚点。

**子维度A：壁垒高度（当前不可替代性）— 权重60%**
- ★★★★★ 三重封锁（技术+认证+专利），3年内无挑战者，且至少满足2个定量指标
- ★★★★☆ 壁垒高+认证周期2年+，且至少满足1个定量指标
- ★★★☆☆ 有壁垒但有2-3家竞争者
- ★★☆☆☆ 主要在规模/渠道，可被资本复制
- ★☆☆☆☆ 标准化充分竞争

**定量锚点（新增，用于佐证定性判断）：**
| 锚点 | ★★★★★佐证 | ★★★☆☆佐证 | 用途 |
|-----|----------|----------|------|
| 专利授权量 | 发明专利>200项且仍在增长 | 专利<50项或持续下滑 | 技术护城河量化 |
| 客户认证 | 通过海外头部客户认证+批量供货 | 仅有国内中小客户 | 客户锁定证据 |
| 替代成本 | 客户替换供应商需>12个月 | 替换<3个月或无认证壁垒 | 切换成本估算 |
| 毛利率稳定性 | 近3年毛利率波动<5pct | 毛利率波动>15pct | 定价权强弱 |

**子维度B：壁垒持续性（未来3年能否维持）— 权重40%**
- ★★★★★ 持续高研发投入(研发费用率>15%)+专利持续增长+新竞争者进入门槛持续提高
- ★★★☆☆ 有维持能力但研发费用率5-15%之间，可能有新进入者
- ★☆☆☆☆ 研发费用率<5%，壁垒正在被侵蚀

**综合围墙评分 = 高度×0.6 + 持续性×0.4**

**否决条件：** 没有任何技术壁垒证据（专利/认证/cost优势/客户锁定全无）→ **一票否决**

**⚠️ 更新：国产化率与围墙的关系修正（重要优化）**
- **原规则：** 国产化率>70%的环节，围墙评分自动上限为★★★★。**逻辑有误。** 国产化率高可能有两种含义，应区分：
  - 国产化率从低到高是**本土龙头主导的替代**（如光伏、动力电池）→ 龙头已建立新壁垒（规模+成本+客户锁定），围墙评分**不自动上限**
  - 国产化率从低到高是**外企技术扩散+多家竞争**（如谐波减速器竞争白热化）→ 壁垒确实在降低
- **新规则：** 国产化率>70%时，检查该环节头部企业的毛利率是否>35%且稳定：
  - 毛利率>35%且稳定 → 新壁垒已形成，不自动上限
  - 毛利率<35%或下滑趋势 → 竞争恶化，围墙评分上限★★★★

**否决条件汇总（任何一个触发，该环节自动评为"暂不考虑"）：**

| 否决条件 | 适用维度 | 触发说明 |
|---------|---------|---------|
| 营收CAGR<10% | 增长 | 成长逻辑不存在 |
| 毛利率<20% | 利润 | 护城河太薄或纯价格战 |
| 无任何技术壁垒证据 | 围墙 | 无卡脖子属性，不符合框架前提 |
| 营收增速连续2季环比下降 | 增长（减速条款） | 成长趋势被破坏 |
| 连续3季毛利率下滑 | 利润（减速条款） | 护城河正在变薄 |

**否决触发后续动作：**
- 触发否决的环节自动标记 `否决触发: true`
- 从配置清单中移除，转入"暂不考虑"
- 在反思记录中记录否决原因
- 若后续数据变化（如毛利率从18%修复到25%），下次拆解时可重新评估

**综合 = 增长×0.35 + 利润×0.35 + 围墙×0.30**

| 综合分 | 评级 | 仓位建议 |
|--------|------|---------|
| ≥4.5 | 核心重仓区 | 哑铃核心仓位 |
| 3.5-4.4 | 卫星配置区 | 哑铃卫星仓位 |
| 2.5-3.4 | 观察名单 | 不入仓，跟踪 |
| <2.5 | 暂不考虑 | 跳过 |

**特殊规则：** 围墙极高(★★★★★)但增速暂时仅★★★时，可据政策优先级手动上调一档。

#### 评分置信度

每次评分同时输出**置信度**（高/中/低），在后续Phase 4A对抗验证后将根据辩论结果更新。

| 置信度 | 含义 | 默认触发 |
|--------|------|---------|
| 高 | 多源数据交叉验证一致，结论可靠 | 3年以上公开财务数据+明确龙头地位 |
| 中 | 部分指标依赖估算或定性判断 | 次新股/数据不完整/行业早期 |
| 低 | 主要基于定性判断，财务数据有限 | 初创期/无独立财务数据/技术路线未定 |

### Phase 4A：对抗验证（多空辩论）

> **定位：** 在三高评分初步完成后，对评分≥3.5（卫星配置以上）的环节进行多空对抗验证，检验评分结论的稳健性。
> **目标：** 不是推翻评分，而是暴露每个环节的盲点风险，最终输出"置信度调整"和"关键风险清单"。

#### 4A.0 辩论效率分级（优化新增）

> **问题：** 原设计对所有评分≥3.5的环节执行同等深度的辩论，导致大量token消耗在低价值环节。
> **解决：** 按评分区间分级投入辩论资源，高评分环节全量辩论，中评分环节精简辩论，低评分环节不辩论。

| 评分区间 | 辩论模式 | 说明 | Token消耗对比 |
|---------|---------|------|-------------|
| ≥4.5（核心重仓区） | 🟥 **全量辩论** | 完整双流（多头+空头+裁决），严格独立撰写 | 基准100% |
| 3.5-4.4（卫星配置区） | 🟧 **精简辩论** | 单Agent同时论证两面，结论要求列出最大风险和反驳 | 约40% |
| 2.5-3.4（观察名单） | ⬜ **不辩论** | 仅记录关键风险到反思，不投入辩论资源 | 0% |
| <2.5 | ❌ 跳过 | 否决触发，不分析 | 0% |

**🟧精简辩论的执行提示：**
```
角色A（兼任多空）：同时论证"为什么这个环节值得卫星配置"和"为什么它可能不值得"。
避免"先扬后抑"或"先抑后扬"的叙事套路——用两个独立列表面呈现：
【支持卫星配置的理由】... 【不支持的理由（风险点）】...
结论偏向风险更可能发生的一方，而非多方更强的一方。
```

#### 4A.1 辩论触发条件（更新）

| 条件 | 说明 |
|------|------|
| 默认触发 | 三高评分≥4.5（全量）/ 3.5-4.4（精简）的所有环节 |
| 强制触发 | 评分≥4.5（核心重仓区）或有任何否决条件边缘触发的环节 → 全量辩论 |
| 可跳过 | 评分<3.5且数据完全公开、行业共识明确的环节 |

#### 4A.2 多头论证（Bull Case）

**⚠️ 双流辩论机制（2026-07-03 新增）：**

多头和空头论证必须**独立分步撰写**，禁止先写一方再"顺便"写另一方，防止确认偏误污染论据。

**正确做法：**
1. 先写多头论证，完全不考虑任何空头论据，假设多头前提全部成立
2. 清空上下文，重新以空头身份独立撰写空头论证
3. 最后裁决者同时阅读两份独立报告后做出裁决

**禁止做法：**
- ❌ 多头论证中"顺便指出风险"（那是空头的job）
- ❌ 空头论证中"但长期来看还是看好"（那是多头的job）
- ❌ 同一段内先扬后抑/先抑后扬（把对抗写在了一起）

**Prompt骨架：**
```
你是「{环节名称}」产业链多头研究员。你的任务是：
基于以下材料，论证为什么这个环节值得投资：

{三高评分结果}
{政策评级(来自Phase 1)}
{BOM拆解数据(来自Phase 3)}
{当前国产化状态🟢🟡🔴}

重点论证方向：
1. 国产替代空间——当前国产化率__%，参考海外龙头营收天花板__
2. 技术路线确定性——为什么现有技术路线不会被跳过
3. 客户锁定效应——认证周期、替换成本、供应链粘性
4. 政策催化预期——未来12-18个月可预期的政策落地
5. 竞争格局——为什么后来者难以进入

输出格式：结构化论述，分点但不限篇幅。
```

#### 4A.3 空头论证（Bear Case）

**Prompt骨架：**
```
你是「{环节名称}」产业链空头研究员。你的任务是：
基于以下材料，论证为什么这个环节存在风险：

{三高评分结果}
{政策评级(来自Phase 1)} 
{BOM拆解数据(来自Phase 3)}
{当前国产化状态🟢🟡🔴}

重点质疑方向：
1. 技术路线风险——现有路线是否可能被下一代技术替代
2. 替代进度不及预期——客户认证"延后"而非"已交付"的信号
3. 竞争格局恶化——国内竞争者数量快速增加、毛利率趋同
4. 估值透支——当前PS是否已price-in了过多的替代预期
5. 政策兑现风险——政策是否只是"鼓励"而非"强制"

输出格式：结构化论述，直接引用多头论点进行反驳。
```

#### 4A.4 裁决输出

多头和空头各完成一轮论证后，由裁决者（复用AI角色）输出结构化裁决：

```json
{
  "环节名称": "RV减速器",
  "辩论结论": {
    "多头胜出": true,
    "置信度调整": 0,
    "发现关键风险": "量产良率爬坡速度可能低于预期",
    "风险等级": "🟡中等",
    "建议动作": "维持评分，在仓位管理中加入良率监控触发器"
  },
  "置信度更新": {
    "原置信度": "高",
    "新置信度": "高",
    "调整原因": "多空双方均未动摇核心判断"
  }
}
```

**置信度调整规则：**

| 辩论结果 | 置信度调整 | 对应动作 |
|---------|-----------|---------|
| 多头明确胜出，空头未提出有效风险 | +0，维持 | 正常配置 |
| 多头胜出，但空头发现有效风险点 | +0，但标注风险 | 加入监控清单 |
| 势均力敌，无法判定 | 降一档（高→中/中→低） | 降级为观察，等更多数据 |
| 空头明确胜出 | 降两档 | 暂不入仓，标记为"待验证" |

#### 4A.5 辩论结果在框架中的传导

辩论的产出（置信度更新 + 关键风险清单）向后传送到以下环节：

| 框架下游环节 | 辩论产出的用途 |
|-------------|--------------|
| 哑铃仓位计算 | 置信度"低"的环节分配更小的初始仓位 |
| 逻辑休眠触发 | 辩论中发现的关键风险点加入监听清单 |
| 倒三角建仓 | 置信度"中"的环节需要更保守的PS锚定 |
| 季度健康检查 | 辩论结论作为下次检查的基线对比 |

### Phase 6：知识积累与反思

#### 6.1 基础记忆更新

拆解完成后，执行以下记忆更新：

1. 在 MEMORY.md 中新增或更新该产业链的BOM图谱索引
2. 将关键结论（政策评级、推荐清单、综合评分表、辩论裁决结果）写入相应的 memory 文件
3. 在文件的 metadata 中记录拆解日期，以便后续做时间序列跟踪

#### 6.2 决策反思（Reflection）

每次完成BOM拆解+对抗验证后，自动生成一条**反思记录**，追加到该产业链的记忆文件中：

**反思Prompt骨架：**
```
你正在复盘一次产业链BOM拆解。基于以下材料，输出2-4句简短反思：

{本次拆解结论摘要}
{本次辩论裁决摘要}
{上一次同产业链拆解的结论（如有）}

如果是首次拆解，回答：
  - 本产业链最关键的3个未知因素是什么？
  - 下次复查时重点验证什么？

如果是重复拆解，回答：
  - 相比上次，结论有哪些变化？
  - 上次的判断哪些对了/错了？
  - 1条具体教训用于下次拆解。
```

**反思记录格式：**
```
📌 BOM反思 | {产业链名称} | {YYYY-MM-DD}
- 关键变化：...
- 正确判断：...
- 错误判断：...
- 教训：...
- 下次验证重点：...
```

#### 6.3 时间序列知识积累

产业链的每次拆解+反思按时间顺序积累，形成可追溯的判断演化史：

```
产业链：人形机器人 — RV减速器
═══════════════════════
[2026-01-15] 首次拆解
  评分：4.2（卫星配置）
  关键风险：量产良率
  反思：等待绿的谐波Q2良率数据

[2026-04-10] 第二次拆解（季度复查）
  评分：4.5（核心重仓↑）
  变化：良率环比提升12pct，国产化率从18%→27%
  反思：良率爬坡速度超预期，确认升级
  
[2026-07-03] 第三次拆解
  评分：...
```

**作用：** 时间序列表让框架能回答"这个判断的变化趋势是什么"而不是只知道当前快照。当某个环节评分从4.5持续降到2.5时，无需等待否决条件触发即可预警。

#### 6.4 反思回注机制

下次拆解同一条产业链时，注入以下历史上下文：
- 最近3次反思记录（供AI Agent参考）
- 上次辩论中发现但未解决的关键风险（作为本次验证起点）
- 上次置信度与后续实际走势的对比（自我校正）
